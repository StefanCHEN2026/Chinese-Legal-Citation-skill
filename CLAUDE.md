# 法学论文脚注引注格式核查与修订 Skill

## 概述

此 Skill 用于核查法学论文 .docx 文件中脚注的引注格式是否符合《法学引注手册》规范，并以 Word 修订模式（Track Changes）进行格式修正，同时添加蓝色批注说明修改原因。

**只修改格式，不修改内容。**

## 支持的语言

- 中文引注（含各类中文文献）
- 英文引注

不支持日文、法文、德文等其他语言的引注核查。

## 工作流程

当用户请求核查某篇法学论文的脚注时，按以下步骤操作：

### 第一步：提取脚注

运行解析工具，将脚注导出为 JSON：

```bash
python scripts/docx_parser.py <论文.docx> -o footnotes.json
```

### 第二步：逐条核查

读取 `footnotes.json`，对每条脚注按以下子步骤处理：

#### 2a. 判断是否为引注

脚注分为"引注类"和"说明类"两种：
- **引注**：包含文献出处信息，需要核查格式
- **说明**：纯文字解释或补充说明，不需要核查格式，跳过

判断方法：脚注中包含作者名、文献名、出版社、期刊名、链接等信息的是引注；纯粹的文字说明、背景介绍、统计数据（有明确文献出处的除外）、案号引用、法律法规条文引用则属于说明性脚注。

**重要**：引注类脚注中也常混有说明性文字。需要**先分割**，只对引注部分进行格式核查，保留说明部分不改。

#### 2b. 判定引注语言

对每条引注，按以下规则判定语言：

- **中文引注**：包含 ANY CJK 字符（U+4E00-9FFF, U+3400-4DBF, U+F900-FAFF）
- **英文引注**：不含 CJK 字符，以英文字母开头

> 日文、法文、德文等其他语言仅提示"非中英文引注，暂不支持核查"，跳过。

#### 2c. 判定引注类型

**中文引注类型**（按优先级判定）：

1. **网络文献** (`internet`)：含 HTTP/HTTPS 链接、微信公众号、博客、App
2. **再次引用** (`repeated`)：以"同前注""同上注""同上"开头
3. **司法案例** (`case`)：含"诉""案"+案号/判决书，或"人民法院""指导案例""公报"
4. **法律文件** (`legal`)：含《×法》《×条例》《×办法》《×规定》或文号（法释、国发等）
5. **古籍** (`ancient`)：作者前有（朝代）标记，或含卷+刻本/影印
6. **学位论文** (`dissertation`)：含"博士学位论文""硕士学位论文"
7. **报纸文章** (`article_newspaper`)：含"载《》"+年月日
8. **文集文章** (`article_collection`)：含"载"+主编/编/文集/论丛
9. **期刊文章** (`article_journal`)：含"载《》"+年份期次，或"年第X期"
10. **图书** (`book`)：含"出版社"+"年版"或"第X版"
11. **未知** (`unknown`)：无法判定，通用规则检查

判定依据详见 `references/reference-index.md`。

**英文引注类型**（按优先级判定）：

1. **网络文献** (`internet`)：含 HTTP/HTTPS 链接、"visited on"
2. **再次引用** (`repeated`)：以"Ibid."开头，或含"supra note"
3. **司法案例** (`case`)：含"v."、U.S. + 数字、ICJ、ICSID、UKSC 等
4. **期刊文章** (`article_journal`)：含"Vol."、期刊特征词
5. **图书** (`book`)：含"Press""Publishing""ed."等

判定依据详见 `scripts/citation_checker.py` 的 `classify_english_citation()` 函数逻辑。

#### 2d. 查找对应规则

根据上一步判定的引注类型和语言，读取对应的规则文件：

| 类型 | 规则文件 |
|------|---------|
| 通用规则 | `references/rules-general.md` |
| 中文-图书 | `references/rules-chinese-books.md` |
| 中文-期刊/报纸/文集文章 | `references/rules-chinese-articles.md` |
| 中文-网络文献 | `references/rules-chinese-internet.md` |
| 中文-未发表文献 | `references/rules-chinese-unpublished.md` |
| 中文-法律文件 | `references/rules-chinese-legal.md` |
| 中文-司法案例 | `references/rules-chinese-cases.md` |
| 英文-全部 | `references/rules-english.md` |
| 再次引用 | `references/rules-repeated-citation.md` |

#### 2e. 逐条核查格式

将引注文本与规则文件中的每条检查项核对，重点检查以下几类问题：

**中文常见错误**（来自 `reference-index.md` 反查表）：
- 作者名后缺冒号
- 书名未用书名号《》
- 缺少"载"字
- 缺少出版年份
- 页码格式缺"第"和"页"
- 译作缺国籍或译者
- 国籍标记括号错误（应用方括号[]，非圆括号或六角括号）
- 网络文献缺上传/访问日期
- 法律文件书名号误用
- 文号年份括号错误（应用六角括号〔〕）
- 案例案号格式错误
- 案例缺案例来源
- 中英文标点混用
- 报纸版号写"02版"

**英文常见错误**（来自 `reference-index.md` 反查表）：
- 英文书名未用斜体
- 英文期刊缺卷号
- 英文引注缺 See（概括引用）
- 英文"姓,名"格式错误（应用自然顺序）
- 多次引用略写格式错误
- "同上注"误用
- 英文页码 p. 写成 pp.
- 报纸版号写"02版"
- 英文日期格式误用

**核查原则**：
- 只报告格式错误，不质疑内容（如文献名、出版社名、作者名是否正确）
- 如果某项规则手册未明确规定，不强行要求
- 同一引注可能违反多条规则，全部列出
- 标注对应手册条款编号

### 第三步：生成修正数据

对每条发现的错误，生成一条修正记录。格式见 `CORRECTIONS_SCHEMA.md`。

**核心字段**：

```json
{
  "footnote_id": "3",
  "rule": "第96条",
  "error_type": "作者格式错误",
  "old_text": "Yan and Hyman",
  "new_text": "Yan & Hyman",
  "reason": "两位作者之间应使用 & 而非 and"
}
```

**修正原则**：
- `old_text` 必须是脚注中**实际存在**的文本片段（会影响 docx 中的文本查找替换）
- 只替换格式相关的标点、连接词、标记符号，不修改文献内容
- 如果一处错误涉及多处替换，拆成多条修正

### 第四步：报告核查结果

在应用修改前，先向用户报告核查结果摘要：

```
## 核查结果摘要

- 脚注总数：32
- 引注脚注：28
- 说明性脚注：4（跳过）
- 发现错误：15 处
- 涉及脚注：10 条

### 错误类型分布
- 缺"载"字：3 处
- 页码格式错误：2 处
- 标点符号错误：4 处
- 英文格式错误：3 处
- ...
```

列出每条错误的具体信息，询问用户确认后再执行修改。

### 第五步：备份并应用修订

**安全原则：任何时候都不修改用户原始文件。修订工具始终在副本上工作。**

1. 首先备份原始文件：
```
copy <论文.docx> <论文_备份.docx>
```

2. 用户确认修正清单后，运行修订工具：
```bash
python scripts/docx_revisor.py <论文.docx> -c corrections.json -o <论文_已修订.docx>
```

3. 最终交付两个文件：
   - `论文_备份.docx` — 原始文件备份（首次创建，已存在则跳过）
   - `论文_已修订.docx` — 包含修订模式和蓝色批注的新文件

> **说明**：`docx_revisor.py` 内部实现是先 `shutil.copy2(输入, 输出)` 再修改输出文件，即使不手动备份，原文件也不会被触及。但显式备份更安全、更透明。

### 第六步：输出结果

告知用户：
- 输出文件路径
- 修订数量统计
- 批注数量统计
- 提示用户在 Word 中打开后查看修订模式和批注

## 注意事项

1. **不修改内容**：只修改标点符号、连接词、括号类型、格式标记（如"第X页"格式），不修改文献名称、作者姓名、出版社名称等实质内容。
2. **解释性文字不核查**：脚注中除了引注外的说明性文字（如"参见前文讨论""该观点也有争议"等）不进行引注格式核查。
3. **多文献脚注**：一条脚注含多个引用的，逐个核查每个引注。
4. **外文文献优先中译本**：英文引注中如果存在中译本，按照手册第95条，优先引用中译本。
5. **不确定时提示**：引注类型模糊或不常见的格式，向用户提示，标注"建议人工复核"。
6. **同一错误重复出现**：不单独生成新修正，逐条列出。
